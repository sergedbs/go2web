# go2web

A command-line app that performs HTTP/HTTPS requests over raw TCP sockets (without HTTP client libraries), supports web search, redirects, retries, disk caching, and human-readable output.

## Install (`uv`)

```bash
uv sync
```

## Usage

```bash
uv run go2web -h
uv run go2web -u https://example.com
uv run go2web -u https://example.com --timeout 8 --retries 1
uv run go2web -s sockets over tcp
uv run go2web -s sockets over tcp --engine wikipedia
uv run go2web -s sockets over tcp --open 1
uv run go2web -s sockets over tcp --interactive
uv run go2web -s sockets over tcp --json
uv run go2web --clear-cache
```

## Executable Name vs Package Folder

The project uses `go2web/` as the Python package directory.
The executable is still named `go2web` and is generated by project scripts:

```bash
uv sync
.venv/bin/go2web -h
```

You can also run it without activating venv:

```bash
uv run go2web -h
```

## Features

- HTTP and HTTPS over raw sockets (`socket` + `ssl`)
- Redirect handling (`3xx` with `Location`)
- Chunked and compressed response decoding
- Bounded retry policy for timeout/connect errors
- Search engines:
  - `ddg` (DuckDuckGo HTML)
  - `wikipedia` (Wikipedia API)
- Human-readable rendering:
  - HTML main-content extraction
  - JSON pretty-printing
- Structured search output with `--json`
- Access/open search results from CLI (`--open`, `--interactive`)
- Disk cache with TTL and controls (`--no-cache`, `--cache-dir`, `--clear-cache`)
- Content negotiation via `Accept` header

## Key Options

- `-u, --url <URL>`: fetch a URL and print readable content
- `-s, --search <terms...>`: search and print top 10 results
- `--engine {ddg,wikipedia}`: choose search backend
- `--json`: output search results in stable JSON schema
- `--timeout <seconds>`: request timeout (default `10`)
- `--retries <n>`: retry timeout/connect failures (default `0`)
- `--open <N>`: open the Nth search result
- `--interactive`: choose result via arrow-key UI
- `--cache-ttl <seconds>`: cache TTL (default `600`)
- `--no-cache`: disable cache for current run
- `--cache-dir <path>`: cache directory override
- `--clear-cache`: clear cache entries before execution

## JSON Output Schema (`--json`)

```json
{
  "query": "cats",
  "engine": "wikipedia",
  "generated_at": "2026-04-19T00:00:00+00:00",

      "title": "Cat",
      "url": "https://en.wikipedia.org/wiki/Cat",
      "snippet": "..."
    }
  ]
}
```
